Hoppscotch fournit une variété de fonctionnalités et a été construit de A à Z avec l'accessibilité et la facilité d'utilisation en tête.

<img src="/navigation/preview-light.png" class="light-img" width="1280" height="640" alt=""/>
<img src="/navigation/preview-dark.png" class="dark-img" width="1280" height="640" alt=""/>

L'interface utilisateur de Hoppscotch est composée des éléments suivants :

- La barre supérieure vous permet de visualiser notre répertoire, de vous connecter avec votre compte et présente des liens utiles.

- La barre latérale gauche donne accès à la suite d'outils qui peuvent être utilisés pour le développement web.

- Vous pouvez changer votre langue préférée dans le coin inférieur droit.

> Jetons un coup d'œil à l'interface de tous les outils.

## Plateforme API REST

---

<img src="/navigation/preview-light.png" class="light-img" width="1280" height="640" alt=""/>
<img src="/navigation/preview-dark.png" class="dark-img" width="1280" height="640" alt=""/>

- La zone centrale est l'endroit où vous construisez et travaillez avec les requêtes d'API.
- Dans la barre latérale gauche, sous le diviseur, des icônes vous dirigent vers des volets spécifiques pour les requêtes, les options et les réponses.
- Le panneau de droite vous permet d'afficher l'historique de vos requêtes d'API récentes, les collections et les environnements.

_Voir aussi:_ [plateforme REST](/rest)

### **Historique**

<img src="/navigation/HistoryPanel.png" height="300" width="265" alt="History Sidepanel"/>
Affiche les récentes requêtes d'API que vous avez effectuées, ce qui vous permet de reprendre là où vous l'avez laissé.

### **Collections**

<img src="/navigation/collectionsSC.png"  height="300" width="265" alt="Collections Sidepanel"/>
Hoppscotch vous permet d'organiser vos requêtes d'API avec des collections et des dossiers.

_Voir aussi:_ [L'utilisation des collections](/quickstart/rest#using-collections)

### **Environnements**

<img src="/navigation/EnvironmentSC.png"  height="300" width="265" alt="Environments Sidepanel"/>
Stockez les variables et réutilisez les valeurs dans vos requêtes et scripts. Pour définir l'environnement, sélectionnez dans la liste déroulante des environnements disponibles.

_Voir aussi:_ [L'utilisation des variables d'environnement](/quickstart/rest#environment-variables)

## Éditeur GraphQL

---

<img src="/navigation/graphqlpageSC.png"  width="1280" height="640" alt=""/>

- La zone centrale est l'endroit où vous travaillez avec vos requêtes graphql.
- Dans la barre latérale gauche, sous le diviseur, se trouvent des icônes qui vous dirigent vers des panneaux spécifiques pour les schémas, les requêtes et les réponses.
- Le panneau de droite comporte des onglets pour la documentation des schémas, les requêtes, l'historique et les collections. Ce panneau peut être masqué pour plus de commodité.

_Voir aussi:_ [platforme GraphQl](/graphql)

## Générateur de documentation API

---

<img src="/navigation/APIDocsImportHighlight.png"  width="1280" height="640" alt=""/>
Vous pouvez importer des collections.
Ou choisir parmi les collections existantes que vous avez créées dans Hoppscotch dans le panneau de droite.
Cliquez sur Générer la documentation pour obtenir la documentation de l'API.

_Voir aussi:_ [Documentation des collections](/quickstart/documentation)
